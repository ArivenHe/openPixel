# OpenPixel 线下互动活动策划

## 1. 活动概述

### 1.1 活动名称

OpenPixel 开源精神现场共创活动

### 1.2 活动定位

本活动面向校园线下活动、社团开放日、开源宣讲会、技术文化节等场景，通过手机 H5 与现场大屏的实时互动，让参与者在低门槛任务中理解开源协作中的核心概念：Fork、PR、Star、Commit、Review、CI、Conflict 与社区共创。

活动不要求参与者具备编程基础。系统将抽象的开源流程转化为创意接龙、代码补全、知识分享、积分奖励和现场抽奖，让非技术专业同学也能参与并获得反馈。

### 1.3 活动目标

- 让 200 人以上现场观众通过网站参与互动。
- 用可视化大屏呈现创意共创、代码提交和实时贡献数据。
- 引导参与者完成至少 1 次创意提交、1 次 Star、若干代码补全或知识分享。
- 通过积分和奖品机制提升参与率与留存时长。
- 活动结束后导出创意、代码提交、参与者、知识分享和抽奖记录，形成可复盘的开源文化资产。

### 1.4 适用规模

- 推荐规模：50-300 人。
- 推荐时长：60-90 分钟。
- 推荐设备：1 块主大屏、1 台控制电脑、1 台备用电脑、稳定 Wi-Fi 或校园网。

## 2. 核心体验设计

### 2.1 参与者移动端

参与者通过连接进入移动端 H5，完成学号登记后可参与以下互动：

- 灵感墙：围绕管理员预设话题提交 50 字以内创意方案，模拟一次 PR。
- Star 点赞：浏览他人创意并 Star 支持，每人对同一创意只能 Star 一次。
- 代码补全：完成 JavaScript、Python、Java、Go 等低门槛选择题式代码任务，模拟 Commit 与测试通过。
- 盲盒知识共享：提交一条开源相关理解、经验或想法，完成一次知识贡献。
- 奖励中心：查看当前积分、奖品库存、抽奖资格和中奖状态。

### 2.2 现场大屏

大屏用于制造现场反馈感和集体参与感：

- 灵感开源树：创意提交后实时生成分支节点，Star 越多节点越突出。
- 代码共创墙：实时展示任务完成状态和最新 Commit 流。
- 数据大盘：展示 Active Contributor、Total Commits、Drawable Users 等指标。
- 实时奖品：展示当前奖品和剩余库存。
- MVP & 抽奖：展示最高 Star 创意和抽奖进度。

### 2.3 工作人员后台

工作人员通过管理台完成现场运营：

- 创建、修改、删除灵感墙主干话题。
- 查看参与者积分排行和贡献情况。
- 新增、修改、删除奖品。
- 实时调整奖品库存、所需积分和是否进入随机奖池。
- 导出活动数据。

### 2.4 抽奖环节

活动设置独立抽奖操作页和抽奖展示大屏：

- 工作人员在抽奖操作页输入或选择满足条件的学号。
- 抽奖展示大屏实时展示中奖结果。
- 抽奖历史持久化保存，便于核销和复盘。
- MVP 奖按 Star 数最高的创意自动计算。

## 3. 活动流程

## 4. 积分与奖励规则

### 4.1 积分规则

| 行为                   | 积分   |
| -------------------- | ----:|
| 提交 1 条灵感 PR          | +1 分 |
| 每成功完成 5 次代码补全 Commit | +1 分 |
| 首次完成盲盒知识共享           | +1 分 |

### 4.2 奖品建议

| 奖品              | 建议门槛 | 建议库存 | 说明             |
| --------------- | ----:| ----:| -------------- |
| 开源贡献者贴纸         | 2 分  | 200  | 低门槛普惠奖         |
| OpenPixel 共创者徽章 | 3 分  | 100  | 中等激励           |
| 终极大奖            | 3 分  | 1-3  | 用于压轴抽奖         |
| MVP 奖           | 无需抽奖 | 1    | Star 数最高创意作者获得 |

### 4.3 核销建议

- 抽奖结果以抽奖展示大屏和后台记录为准。
- 工作人员核对学号后发放奖品。
- 已中奖或已核销奖品不要在后台删除，避免记录缺失。
- 若奖品库存临时变化，应由管理员在后台实时调整。

## 7. 技术执行清单

### 7.1 启动方式

本地演示可使用：

```bash
npm run install:all
cd backend
npm run dev
cd frontend
npm run dev
```

Docker 部署可使用：

```bash
docker compose up --build
```

### 7.2 正式活动前检查

- 确认移动端、大屏端、管理台、抽奖页均可访问。
- 确认 `ADMIN_TOKEN` 已设置，且不会展示给普通参与者。
- 确认 Redis 或本地文件持久化可正常工作。
- 确认敏感词审计生效。
- 确认导出接口可用：
  - `/api/export/ideas`
  - `/api/export/code-submissions`
  - `/api/export/participants`
  - `/api/export/blind-box-shares`
  - `/api/export/lottery`
- 确认大屏浏览器已全屏，系统休眠和通知已关闭。
- 确认现场 Wi-Fi 对参与人数有足够承载。

### 7.3 正式活动前清空演示数据

```bash
cd backend
npm run reset:local
```

### 7.4 快速验收

```bash
cd backend
npm run smoke
```

验收链路包括学号登记、创意提交、Star、代码补全、盲盒知识共享、抽奖资格和开奖。

## 8. 主持人口播参考

### 8.1 开场

欢迎大家来到 OpenPixel 开源精神现场共创活动。今天不需要大家提前会写代码，只需要用手机完成几个简单互动，就能体验一次开源社区里的协同创作。你提交的创意会像一个 PR 一样出现在大屏上，别人可以给你的想法 Star；你完成的代码补全会变成一次 Commit；最终积分会进入抽奖池。

### 8.2 灵感墙引导

现在请大家进入灵感墙，选择一个你感兴趣的校园话题，提交一个 50 字以内的解决方案。提交后请抬头看大屏，你的想法会长成一条新的分支。

### 8.3 Star 引导

接下来大家可以浏览其他同学的创意，看到你认可的方案就点 Star。Star 最高的创意会成为今天的 MVP 候选。

### 8.4 代码补全引导

现在进入代码补全挑战。题目是选择题形式，不要求你真正写代码。每完成 5 次正确提交，就可以获得 1 分。大屏会实时展示大家的 Commit。

### 8.5 抽奖引导

请大家打开奖励中心查看自己的积分和抽奖资格。满足条件的同学将进入抽奖池，中奖结果会在大屏实时展示。

## 9. 风险预案

| 风险           | 影响                   | 预案                                   |
| ------------ | -------------------- | ------------------------------------ |
| 现场网络不稳定      | 手机无法访问或 WebSocket 断连 | 准备备用热点；提前测试校园网；大屏与后端尽量走同一局域网         |
| 大屏浏览器卡顿      | 可视化效果延迟              | 关闭无关程序；使用 Chrome；降低其他投屏负载            |
| 管理员 token 泄露 | 后台被误操作               | 活动前更换 token；仅发给工作人员；出现异常立即重启并更换      |
| 恶意刷屏或不当内容    | 影响现场秩序               | 启用敏感词审计和禁言；现场主持人口头提醒；必要时暂停互动         |
| 奖品库存不足       | 现场核销混乱               | 后台实时调整库存；抽奖前确认剩余数量                   |
| 数据误清空        | 复盘材料丢失               | 活动前后分别备份状态文件或 Redis 数据；抽奖前避免执行 reset |
| 参与者不会操作      | 参与率下降                | 设置现场引导员；入口二维码旁放简短规则                  |

## 10. 数据复盘

活动结束后建议导出并整理以下数据：

- 参与人数：登记学号数、活跃贡献者数。
- 创意数据：话题数量、创意数量、最高 Star 创意。
- 代码数据：总 Commit 数、各语言任务完成情况。
- 奖励数据：积分分布、抽奖池人数、中奖记录。
- 内容资产：优秀创意、知识分享内容、现场照片和大屏截图。

复盘产出可用于公众号推文、社团招新材料、开源课程案例或后续活动选题。
